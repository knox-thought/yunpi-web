# กลยุทธ์ลูกค้า — LINE OA ก่อน ไม่ทำล็อกอินเอง

## ตัดสินใจ: LINE Official Account ก่อน, ระบบสมาชิกล็อกอินเอง = ไม่ทำ (ตอนนี้)

| | LINE OA | ระบบล็อกอินเอง |
|---|---|---|
| ลูกค้าไทยหิ้วของ | อยู่บนไลน์อยู่แล้ว แอด = 0 แรงเสียดทาน | ต้องสมัคร/จำรหัส คนไม่ทำ |
| ได้อะไร | broadcast, rich menu, แท็ก, แชต, auto-reply ฟรี | ต้องสร้าง auth/reset/session/security เอง |
| ตรง roadmap | ใช่ (ทั้งระบบวางไว้ให้ AI ผ่านไลน์) | สวนทาง |
| ตอนนี้ | เหมาะ ลงทุนต่ำ | เร็วเกินไป (YAGNI) |

## ขั้นบันได
1. **ตอนนี้:** ตั้ง LINE OA → เก็บลูกค้าเป็นเพื่อน → rich menu (สั่งของ / เช็คสถานะ / ราคา-วิธีส่ง / โปรโมชั่น) + broadcast + แท็ก
2. **ผูกตัวตน:** เก็บ LINE `userId` → `Customer.lineId` ที่มีอยู่แล้วใน ops (ตอนลูกค้าทัก หรือผ่าน LIFF)
3. **บิล/ใบแจ้งวันนี้:** ใช้ลิงก์ `/s/<token>` ของ ops (เปิดดูไม่ต้องล็อกอิน ส่งทางไลน์) — มีแล้ว ไม่ต้องทำใหม่
4. **ทีหลัง (เมื่อจำเป็น):** LIFF mini-app ใน LINE = "เช็คสถานะ/ประวัติของฉัน" → ล็อกอินด้วย **LINE Login** (ไม่มีรหัสผ่าน) ผูก `Customer` → อ่าน ops API. **นี่คือระบบสมาชิก** แต่ขี่บนไลน์
5. **ล็อกอิน username/password เอง:** เกือบจะไม่ต้องมีเลย

## หัวใจ
ระบบสมาชิกล็อกอินเอง = over-engineer. **LINE OA → (ทีหลัง) LINE Login/LIFF** = งานน้อยกว่า อยู่ที่ที่ลูกค้าอยู่ ต่อกับแผน AI-ผ่านไลน์

## ข้อควรรู้
- broadcast ฟรีมีโควตา/เดือน เกินต้องเสียเงิน — ฐาน 1,000+ คนยิงทุกเดือนอาจต้องแพ็กเสียเงิน (ยังถูกกว่าสร้างระบบเอง)
- LIFF ต้องมี LINE Developers channel + ผูกกับ OA
- เว็บ marketing (โปรเจกต์นี้) กับ LINE OA ทำงานคู่กัน: เว็บดึงคนแปลกหน้าจาก search → ปุ่ม CTA แอดไลน์ → OA ปิดการขาย/ดูแล
